(static) modularScale(steps, baseopt, ratioopt) → {string}

Source:
  • helpers/modularScale.js, line 35

Establish consistent measurements and spacial relationships throughout your projects by incrementing up or down a defined scale. We provide a list of commonly used scales as pre-defined variables, see below.

Example
// Styles as object usage
const styles = {
   // Increment two steps up the default scale
  'font-size': modularScale(2)
}

// styled-components usage
const div = styled.div`
   // Increment two steps up the default scale
  font-size: ${modularScale(2)}
`

// CSS in JS Output

element {
  'font-size': '1.77689em'
}
Parameters:
Name Type Attributes Default Description
steps number

The amount of steps to move along the scale. Any integer.
base string <optional>
 1em

The base value to generate the scale from.
ratio number | 'minorSecond' | 'majorSecond' | 'minorThird' | 'majorThird' | 'perfectFourth' | 'augFourth' | 'perfectFifth' | 'minorSixth' | 'goldenSection' | 'majorSixth' | 'minorSeventh' | 'majorSeventh' | 'octave' | 'majorTenth' | 'majorEleventh' | 'majorTwelfth' | 'doubleOctave' <optional>
 perfectFourth

The scale ratio, can either be a number or one of the pre-defined scales from below.
Returns:

The size in ems.

(static) stripUnit(value) → {Number}

Source:
  • helpers/stripUnit.js, line 27

Strip the unit from a given CSS value

Example
// Styles as object usage
const styles = {
  '--dimension': stripUnit(100px)
}

// styled-components usage
const div = styled.div`
  --dimension: ${stripUnit(100px)}
`

// CSS in JS Output

element {
  '--dimension': 100
}
Parameters:
Name Type Description
value string

A string containing a value with its unit.
Returns:

Unitless value.
